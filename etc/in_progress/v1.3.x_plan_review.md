#### P1-1: `async_core.py` プールクリーンアップのログ修正

**場所**: `src/nanasqlite/async_core.py` (L1182-1188)

**現状の問題**:
```python
except AttributeError as e:
    # Programming error: conn is not an apsw.Connection
    logging.getLogger(__name__).error(  # ← ERROR レベルは過剰
        "AttributeError during pool cleanup - possible programming error: %s (conn=%r)",
        ...
    )
```

**コメントが「プログラミングエラー」と言いつつ ERROR レベルで継続** という不整合を修正。

**提案修正**:
```python
except AttributeError as e:
    # Unexpected AttributeError - log and continue cleanup for resilience
    logging.getLogger(__name__).warning(
        "Unexpected AttributeError during pool cleanup: %s (conn=%r)",
        e,
        conn,
    )
```

**影響**: ログ出力のみ。後方互換性への影響なし。  
**工数**: 5 分以内。

### 🟠 Priority 2 — v1.3.x 内で完了推奨

#### P2-1: ロック取得タイムアウト機構 (`lock_timeout` パラメータ)

**背景**: `roadmap_2025.md` フェーズ 2.2「タイムアウト・ロック機構の改善」。`implementation_status.md` でも v1.3.0 対象と記載。

**提案実装**:
```python
# NanaSQLite.__init__ に追加
def __init__(self, ..., lock_timeout: Optional[float] = None):
    self._lock_timeout = lock_timeout
    ...

# _acquire_lock() ヘルパーの新設
def _acquire_lock(self):
    if self._lock_timeout is not None:
        acquired = self._lock.acquire(timeout=self._lock_timeout)
        if not acquired:
            raise NanaSQLiteLockError(
                f"Failed to acquire lock within {self._lock_timeout}s"
            )
    else:
        self._lock.acquire()
```

**影響**: オプトイン設計のため後方互換性は完全に維持。  
**工数**: 半日程度。

#### P2-2: バリデーション機能の基盤 (`add_validator`)

**背景**: `roadmap_2025.md` フェーズ 2.3「バリデーション機能の基盤」。`future_features_proposal.md` 提案 6。

**提案実装** (最小限の基盤):
```python
# NanaSQLite に追加
def add_validator(self, key_pattern: str, validator_fn: Callable[[Any], None]) -> None:
    """キーパターンに対するバリデーター関数を登録する"""
    self._validators[key_pattern] = validator_fn

# __setitem__ 内で呼び出し
def _run_validators(self, key: str, value: Any) -> None:
    for pattern, fn in self._validators.items():
        if fnmatch(key, pattern) or key == pattern:
            fn(value)
```

**影響**: オプトイン設計。デフォルトで `_validators = {}` のため既存動作に影響なし。  
**工数**: 1 日程度。

#### P2-3: 簡易バックアップ/リストア (`backup` / `restore`)

**背景**: `roadmap_2025.md` フェーズ 2.4「自動バックアップ」。`future_features_proposal.md` 提案 9。

**提案実装** (APSW の SQLite バックアップ API を活用):
```python
def backup(self, dest_path: str) -> None:
    """データベースをファイルにバックアップする"""
    import apsw
    dest_conn = apsw.Connection(dest_path)
    with dest_conn.backup("main", self._connection, "main") as b:
        b.step()  # 全ページを一括コピー
    dest_conn.close()

def restore(self, src_path: str) -> None:
    """バックアップファイルからデータベースをリストアする"""
    import shutil
    self.close()
    shutil.copy2(src_path, self._db_path)
    # 再接続
    self._reconnect()
```

**影響**: 新規 public メソッドの追加のみ。後方互換性への影響なし。  
**工数**: 半日〜1 日程度。
